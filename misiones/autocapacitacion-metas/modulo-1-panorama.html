🏗️ ¿Qué es M.E.T.A.S técnicamente?

M.E.T.A.S es una aplicación web estática y offline-first que se distribuye por tres frentes —web, PWA y APK Android— desde un mismo código, y usa Supabase como nube para lo único que de verdad necesita servidor. Esa es la frase maestra: apréndela hasta poder decirla sin leerla y desglosar cada término.

💡
Por qué esta misión existe: es una actividad metalingüística — aprender de lo que hacemos. El objetivo es que puedas explicar tu sistema en todos sus aspectos ante cualquier ingeniero que te interpele: sin memorizar respuestas ajenas, sino entendiendo el porqué de cada decisión. Todo lo que afirma este módulo fue verificado contra el código real del repositorio.

📊 Dos preguntas que revelan la arquitectura

🌐 ¿Dónde vive la app?

En ningún servidor propio: son archivos estáticos (HTML+CSS+JS, sin compilación) servidos por GitHub Pages bajo el dominio propio metas.policastsapien.com (archivo CNAME).

💾 ¿Dónde viven los datos?

Primero en el dispositivo (localStorage): esa es la fuente de verdad. La nube Supabase es copia y extensión — sincroniza dispositivos y alimenta al asistente de padres.

⚙️ La idea clave para defender
Los tres frentes NO son tres desarrollos: un solo código se entrega de tres maneras. Por eso una sola persona puede mantener el sistema completo.

🚚 Los frentes de distribución

🌐 Web + 📱 PWA

El sitio se publica con git push. El service worker (sw.js, hoy v27) guarda copia local: la página se vuelve instalable y abre sin conexión. En cada cambio se sube la versión para renovar los teléfonos.

🤖 APK Android

Capacitor envuelve el mismo código (webDir: www) en un APK con id com.policastsapien.metas, firmado con keystore y con versionCode creciente. Camino: npm run sync → Gradle → firma.

📦 npm run sync
Todo el «proceso de build» del proyecto cabe en un script: copiar el código a www/ y dejar que Capacitor lo empaquete. Cero compilación en el frente web: lo que está en el repo es lo que corre.

☁️ La nube: solo lo que exige servidor

💾 Capa local (fuente de verdad)

localStorage con las claves METAS_*, el outbox con reintentos y el caché del service worker. El alumno puede trabajar un día entero sin señal.

☁️ Capa nube (Supabase)

PostgreSQL como servicio: funciones RPC como única puerta, RLS por fila, rate-limit por IP real y deduplicación por evento_id. Plan Free: costo $0.

🔑 En sus palabras
La clave que aparece en el código es la pública (anon): solo permite lo que RLS y las funciones RPC autorizan. La service_role — la peligrosa — jamás sale del servidor.

👥 ¿Quién usa qué?

🧒 Alumno · 👨‍👩‍👧 Familia

El alumno entra por el index a las 32 misiones con su código de aula (su nombre nunca viaja). La familia usa padres.html, el asistente conversacional, con la clave de familia de su hijo.

👨‍🏫 Maestro · 🏫 Dirección

El maestro entra a la Zona Docente con cuenta PROF-XXXX: Plan de Acción, Mi aula, boleta, Campeonísimo. La dirección usa roles (director/rector/admin) con vistas propias.

La regla transversal es la minimización de datos de menores: códigos y claves en lugar de nombres, en todos los frentes.
